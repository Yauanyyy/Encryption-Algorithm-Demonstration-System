### 算法详细介绍

**历史背景：**
凯撒密码（Caesar cipher）以古罗马统帅尤利乌斯·凯撒（Julius Caesar）命名。凯撒在军事和政府通信中使用过将字母按固定偏移量替换的技术，据记载常用偏移 `3` 来对短消息进行手工加密。该方法旨在为当时的信使传递提供基本的保密性，后来成为研究替换密码与密码分析的教学典范。

**原理：**
凯撒密码是一种最古老、最简单的替换加密技术。加密时，将明文中的每个字母替换为字母表中其后第 N 位的字母，N 就是密钥（偏移量）。解密则是反向操作。

例如，当偏移量为 `3` 时:
- 明文: `A B C`
- 密文: `D E F`

**流程：**
1.  获取明文和整数偏移量 `N`。
2.  遍历明文中的每个字符。
3.  如果字符是字母，则找到它在字母表中的位置 `P`。
4.  计算新位置 `P' = (P + N) mod 26`。
5.  新位置 `P'` 对应的字母即为密文字符。
6.  如果字符非字母（如空格），则保持不变。
7.  解密过程则是 `P' = (P - N + 26) mod 26`。

**安全性分析：**
凯撒密码的安全性**极低**。由于密钥空间极小（仅25个可能的有效偏移量），攻击者可以通过**穷举法**（暴力破解）在瞬间破解密文。它仅适用于教学演示，绝不能用于任何实际的安全通信。
